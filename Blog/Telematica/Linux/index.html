Error de vídeo Ubuntu Mate 20.04 LTS

después de la instalación en AMD.

Fecha : Diciembre 2020

Síntomas:

  1. Pantalla de inicio de sesión o el escritorio congelado o extremadamente lento aún usando el método de ‘nomodeset’.
  2. Pantalla de bienvenida o escritorio intermitente: la pantalla parpadea e incluso cambia a un color oscuro.

 

Si el equipo que usas está pasando por esto, continúa leyendo.

 

Diagnóstico:

 

Estos síntomas son causados por el controlador de la tarjeta de vídeo. Hay controladores de video de NVIDIA para Linux que normalmente no funcionan en ordenadores con tarjetas de video como la GeForce GT 540M e incluso hay sistemas NVIDIA que fallan en otros sistemas operativos como Windows.

 

Ubuntu Mate 20.04 LTS decide, en la instalación y por defecto, instalar un controlador privativo o terciario de NVIDIA para ejecutar el sistema operativo. Estos controladores suelen ser totalmente ineficientes: presentan fallas y congelan o ralentizan el sistema.

 

Introducción

 

Me tarde un día completo en encontrar la solución a mi problema especifico, voy a mencionar paso a paso considerando varios escenarios, y en la situación más critica que fue la miá, recomiendo ampliamente el uso de Linux, ya que a pesar de ser complicado al inicio, con una sana y buena instalación tiene numerosas ventajas respecto a otros sistemas operativos.

 

 

Después de la actualización a Ubuntu Mate 20.04 la pantalla se veía así, la solución que voy a proponer parece algo confusa pero si esas opciones no funcionan, entonces tenemos que formatear nuestro sistema operativo.

 

Primero ingrese a la terminal con CTRL + ALT + T sin ver y despues de loggearme con mi usuario y contraseña SUDO SU cambie la contraseña de root para activarla, en caso de que ya puedas ingresar sin problemas con permisos de root esté paso no es necesario. El comando es el siguiente:

 

sudo passwd root

 

Nos pedira ingresar una contraseña para el usuario root y repetirla. Ingresamos a la vista en modo terminal con CTRL + ALT + F6 en mi caso. Ahora nos logueamos como usuario root . Otro ingreso a consola podría ser en modo "recovery mode"

 

Método 1

 

 

Nota: es necesaria una conexión a la red (internet).

 

  1. 1. Reiniciamos la computadora y en mi caso la combinación de teclas es SHIFT + F12 , después ingresamos en modo recovery, y finalmente como se ve en la imagen consola root.

    2.  

  2. 2. Una vez en la consola, ingresar el usuario y la contraseña si es necesario.

    3.  

  3. 3. Tener presente y observar la configuración gráfica y los controladores (drivers) con el siguiente comando:

 

sudo ubuntu-drivers devices

 

Ejemplo, en mi caso, la máquina muestra:

 

> == /sys/devices/pci0000:00/0000:00:01.0/0000:01:00.0 ==
> modalias : pci:v000010DEd00000DF4sv00001028sd0000050Ebc03sc00i00
> vendor : NVIDIA Corporation
> model : GF108M [GeForce GT 540M]
> driver : nvidia-340 - distro non-free
> driver : nvidia-driver-390 - distro non-free recommended
> driver : xserver-xorg-video-nouveau - distro free builtin

 

4. Desinstalar los controladores propietarios, privativos y terceros de NVIDIA y eliminar los controladores restantes.

 

sudo apt-get remove nvidia-*
sudo apt-get autoremove --purge nvidia-*

 

5. Detener el servicio 'lightdm' que ejecuta el administrador de pantalla que inicia el ‘X servers’, las sesiones de usuario y la pantalla de inicio de sesión.

 

sudo service lightdm stop
sudo service lightgm stop

 

6. Verificar la eliminación de todas las librerías no usadas y actualizar.

 

sudo apt-get autoremove
sudo apt-get autoclean
sudo apt-get update
sudo apt-get upgrade

 

7. Instalar el controlador nouveau predeterminado ( xserver-xorg-video-nouveau ). Este controlador funciona en Linux para tarjetas NVIDIA como la GeForce GT 540M:

 

sudo apt-get install xserver-xorg-video-nouveau

 

En caso de que no funcione la instalación de está forma, puede buscar el paquete en github e instalarlo como se muestra a continuación clonando e instalando el paquete:

 

sudo apt install git
git clone https://github.com/gnab/rtl8812au.git
sudo dkms add ./rtl8812au
sudo dkms build rtl8812au/4.2.2 (si obtiene un error que dice que la carpeta 'rtl8812au' no existe, cambie rtl8812au a '8812au')
sudo dkms install rtl8812au/4.2.2 (si obtiene un error que dice que la carpeta 'rtl8812au' no existe, cambie rtl8812au a '8812au')

 

Cargamos el controlador

 

sudo modprobe rtl8812au (o 8812au si recibe un mensaje de error que dice que rtl8812au no existe)

 

8. Verifica en la consola que la instalación está correcta y sin errores.

 

9. Reinicia

 

Reboot

 

Nota: en la página de NVIDA, AMD, etc. hay controladores para descargar y probar directamente con tu tarjeta

 

Resumen:

 

Abrir consola/terminal.
sudo ubuntu-drivers devices
sudo apt-get remove nvidia-*
sudo apt-get autoremove --purge nvidia-*
sudo service lightdm stop
sudo apt-get autoremove
sudo apt-get update
sudo apt-get install xserver-xorg-video-nouveau
Exit/Reboot

 

Método 2

Desintalar e instalar el Escritorio. Primero lo desinstalamos.

 

sudo apt-get remove mate-desktop

 

Para realizar una instalación mínima (sólo la interfaz):

 

sudo apt-get install mate-core

 

Para instalar todo el entorno (incluye aplicaciones):

 

sudo apt-get install mate-desktop-environment

 

Método 3

Parche recomendado en foros de reddit, pero es probable que solo funciones para Xubuntu 20.04.

 

sudo apt-get update
sudo apt-get install xfconf
xfconf-query -c xfwm4 -p /general/vblank_mode -t string -s "glx" --create

 

Método 4

Si ésto nos ocurre después del 22 de abril de 2021 también podemos intentar actualizar a la versión 21.04 desde terminal:

 

sudo apt-get install update-manager
sudo update-manager –devel-release
Precionamos ALT + F2

 

¡Ya está solucionado el problema de vídeo! que la deidad de su preferencia os acompañe.
