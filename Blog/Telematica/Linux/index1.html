Forzar en Ubuntu

la instalación de WineHQ

Fecha : Enero 2021

¿Qué es wine?

 

Wine es un emulador de aplicaciones windows en sistemas operativos linux permite a los usuarios ejecutar aplicaciones de Windows en Linux y otros sistemas operativos tipo Unix. Para ser un poco más técnico, Wine es una capa de compatibilidad; traduce las llamadas del sistema de Windows a Linux y utiliza algunas bibliotecas de Windows, en forma de archivos .dll .

 

Para aquellas personas que vienen migrando de Linux muy posiblemente necesiten de algun software o juego de Windows que no está disponible o no tiene equivalente en Linux. Wine hace posible ejecutar esos programas y juegos de Windows en su escritorio Linux.

 

Es importante reconocer el tipo de versión que vamos a instalar ya que la versión estable tiene menos errores y mayor estabilidad, pero admite menos aplicaciones de Windows. La versión de desarrollo ofrece una mayor compatibilidad, pero tiene más errores no resueltos.

 

¿Por qué en algunas versiones de linux es necesario forzar la instalación de WineHQ?

Porque varias versiones de linux tienen dependencias con la instalación de wine, necesitamos hacer uso de synaptic ya que si esté metodo no funciona necesitamos intentar la instalación de winehq desde synaptic para que nos muestre el nombre de los archivos con que winehq tiene dependencias no compatibles.

 

Instalación de WineHQ

 

Abrimos una terminal con CTRL + ALT + T. El primer paso será habilitar la arquitectura de 32 bits, que aun que nuestro sistema sea de 64 bits, nos será de utilidad más adelante:

 

sudo dpkg --add-architecture i386

 

Ahora debemos de importar las llaves y añadirlas al sistema con este comando:

 

wget -nc https://dl.winehq.org/wine-builds/Release.key

sudo apt-key add Release.key

 

Añadimos el repositorio al sistema, para ello escribimos en la terminal:

 

sudo apt-add-repository https://dl.winehq.org/wine-builds/ubuntu/

 

Después de esté paso es probable que la llave del repositorio esté desactualizada por lo tanto escribimos el siguiente comando para consultar a un servidor seguro de Ubuntu y actualizar la clave pública del repositorio con una válida (Debemos cambiar la clave por la que nos envía el error en la terminal):

 

sudo apt-key adv --keyserver keyserver.ubuntu.com --recv-keys 1F3045A5DF7587C3

 

Ahora agregamos el repositorio

 

sudo apt-add-repository 'deb https://dl.winehq.org/wine-builds/ubuntu/ artful main'

 

En esté paso WineHQ tiene varias dependencias con varios paquetes y versiones por lo que intentamos solucionarlas intalandolas manualmente (podemos intentar saltarnos este paso, y validar que nos envía un error):

 

sudo apt-get install libgnutls30:i386 libldap-2.4-2:i386 libgpg-error0:i386 libxml2:i386 libasound2-plugins:i386 libsdl2-2.0-0:i386 libfreetype6:i386 libdbus-1-3:i386 libsqlite3-0:i386

sudo apt-get update -y

sudo apt-get install -y libsdl2-2.0-0

wget https://download.opensuse.org/repositories/Emulators:/Wine:/Debian/xUbuntu_18.04/amd64/libfaudio0_19.07-0~bionic_amd64.deb

wget https://download.opensuse.org/repositories/Emulators:/Wine:/Debian/xUbuntu_18.04/i386/libfaudio0_19.07-0~bionic_i386.deb

sudo dpkg -i libfaudio0_19.07-0~bionic_amd64.deb libfaudio0_19.07-0~bionic_i386.deb

 

En caso de errores o paquetes rotos antes de reintentar la intalación usamos los comandos:

 

sudo dpkg --configure -a

sudo apt --fix-broken install

 

Finalmente, solo debemos de escribir el comando para instalar la versión estable de Wine:

 

sudo apt-get install --install-recommends winehq-stable

 

Ahora también tenemos acceso a la rama de desarrollo de Wine, el cual contiene muchas más características y mejoras, el problema al ser una versión de desarrollo es que corremos el riesgo de tener algunas fallas con la ejecución. Pero si lo deseas instalar, la que actualmente está en curso ejecutamos:

 

sudo apt-get install --install-recommends winehq-devel

 

Hecha la instalación solo debes de ejecutar este comando para comprobar que se instaló con éxito y saber que versión tienes instalada:

 

wine --version

 

Donde sí fue la versión estable recibiras una respuesta asi:

 

wine-3.6

 

Nota: En caso de que sigan existiendo dependencias y/ó errores, podemos buscar en synaptic winehq-devel nos enviará la versión recomendada, winehq-devel-amd64, winehq-devel-i386, etc., y al intentar la instalación generalmente nos indicara el paquete que hace falta instalar.

 

¡Ya tenemos wine instalado!

 

Desinstalar Wine

 

Desinstalar la versión estable:

 

sudo apt purge winehq-stable

sudo apt-get remove wine-stable

sudo apt-get autoremove

 

Desinstalar la versión de desarrollo:

 

sudo apt purge winehq-devel

sudo apt-get remove wine-devel

sudo apt-get autoremove
